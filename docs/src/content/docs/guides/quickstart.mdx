## 1. Install

<Tabs syncKey="pkg">
  <TabItem label="npm">
    ```bash
    npm install @ogabrielluiz/patchflow
    ```
  </TabItem>
  <TabItem label="pnpm">

  </TabItem>
  <TabItem label="bun">
    ```bash
    bun add @ogabrielluiz/patchflow
    ```
  </TabItem>
  <TabItem label="yarn">
    ```bash
    yarn add @ogabrielluiz/patchflow
    ```
  </TabItem>
</Tabs>

patchflow ships ESM + CJS builds and its own TypeScript declarations. Its only
runtime dependency is [dagre](https://github.com/dagrejs/dagre).

## 2. Render a patch

```ts title="render.ts"
import { render } from '@ogabrielluiz/patchflow';
import { writeFileSync } from 'node:fs';

const svg = render(`
- Oscillator (Out) -> Filter (In)
- Envelope (Out) >> Filter (Cutoff)
- Filter (Out) -> VCA (In)
- LFO (Out) >> VCA (CV)
`);

writeFileSync('patch.svg', svg);
```

That's it. You'll get an SVG you can drop into a README, a blog post, or a PDF.

## 3. Switch to the dark theme

```ts
import { render, darkTheme } from '@ogabrielluiz/patchflow';

const svg = render(notation, { theme: darkTheme });
```

Or override just the bits you care about:

```ts
const svg = render(notation, {
  theme: {
    cable: {
      colors: {
        audio: { stroke: '#ff5f7a', plugTip: '#d94963' },
      },
    },
  },
});
```

The theme object is deep-merged into the default — you never have to specify a
full theme unless you want to. See the [Theming guide](../theming/) for the
full surface.

## 4. Try it live

Head to the [Playground](../../playground/) to edit notation and see the SVG
update in real time. Swap presets, toggle themes, copy the SVG to your
clipboard, or download it as a file.
